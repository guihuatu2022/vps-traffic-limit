## 第 3 步：SSH 端口检测

⏳ 正在检测 SSH 端口...

┌─────────────────────────────────────────────┐
│  ✅ 检测到 SSH 端口:                          │
│     - 22 (从 /etc/ssh/sshd_config)           │

│  ❓ 是否使用以上端口? (Y/n):                  │

│       ├─ 输入 "2222"
│       │   └─ ✅ SSH_PORTS=[2222]
│       ├─ 输入 "22,38030"
│       │   └─ ✅ SSH_PORTS=[22, 38030]
│       ├─ 输入 "abc" (无效)
│       │   └─ ⚠️ "端口号必须是数字，请重新输入"
│       │   └─ 返回端口输入
│       └─ 输入 "0" / "65536" (超出范围)
│           └─ ⚠️ "端口范围 1-65535，请重新输入"
│           └─ 返回端口输入
│
└─ 传参 --ssh-port 38030 (静默模式)
    └─ ✅ SSH_PORTS=[38030]

SSH 端口检测逻辑展开:
┌─ 读取 /etc/ssh/sshd_config
│   ├─ 找到 Port 22 → 加上
│   ├─ 找到 Port 2222 → 加上
│   ├─ 找到 Port 38030 → 加上
│   ├─ 多行 Port → 全部加上
│   │   ⚠️ "检测到多个 SSH 端口: 22, 38030"
│   ├─ 无 Port 指令 → 默认 [22]
│   │   ⚠️ "未找到 Port 配置，使用默认端口 22"
│   └── 文件不存在 → 默认 [22]
│       ⚠️ "sshd_config 不存在，使用默认端口 22"

## 第 6 步：流量计费方向选择

┌─────────────────────────────────────────────┐
│  📊 选择流量统计方向                           │
│                                              │
│  1) egress 上行（出站）[默认 🏆]             │
│     适用: GCP, AWS, Azure, 阿里云, 腾讯云    │
│     这些厂商按"出站流量"计费                   │
│                                              │
│  2) total 双向（合计）                       │
│     适用: Hetzner, BuyVM, OVH, Contabo       │
│     这些厂商按"总流量"计费                    │
│                                              │
│  3) ingress 下行（入站）                     │
│     适用: 极少厂商收下行费                    │
│                                              │
│  提示: 选错方向会导致误判或漏判，请确认厂商规则  │
│                                              │
│  请输入数字 (默认 1):                         │
└─────────────────────────────────────────────┘

📝 用户输入:
├─ Enter / "1" / "egress"
│   └─ ✅ DIRECTION=egress
│   └─ ⚠️ "已选择上行计费 - 适用于 GCP/AWS/Azure/阿里云"
│
├─ "2" / "total"
│   └─ ✅ DIRECTION=total
│   └─ ⚠️ "已选择双向计费 - 适用于 Hetzner/BuyVM/OVH"
│   └─ ⚠️ "请确认你的 VPS 厂商确实按总流量计费"
│   └─ ❓ "确认使用 total 模式?(Y/n)"
│       ├─ Y → 继续
│       └─ n → 返回方向选择
│
├─ "3" / "ingress"
│   └─ ✅ DIRECTION=ingress
│   └─ ⚠️ "⚠️ 下行计费极少见，你确定吗？(y/N)"
│       ├─ y → 继续
│       └─ N / Enter → 返回方向选择
│
└─ 传参 --direction egress (静默模式)
    └─ ✅ 直接设置

```
[2026-05-16 13:06:00] 📊 当前月上行: 45.23 GB / 170 GB
```

### 到达 80% 预警线（136 GB）
```

```
→ 如果配置了通知 → 发送预警

### 到达 95% 预警线（161.5 GB）
```
[2026-05-22 14:33:00] ⚠️ 月上行已达 162.10 GB / 170 GB（95%）请注意！
```
→ 如果配置了通知 → 发送预警

### 超限锁定
```
[2026-05-25 09:15:00] 📊 当前月上行: 171.05 GB / 170 GB
[2026-05-25 09:15:00] ⛔ 月上行超 170 GB，已锁定网卡 ens4 出站
```
→ 写入规则 + 创建 lock file + 发送通知

### 手动解锁后
```
[2026-05-26 10:00:00] ✅ 用户已手动解锁网卡
```

### 跨月自动解锁
```
[2026-06-01 00:03:00] ✅ 新月份已到，解锁网卡 ens4 出站
```


```bash
sudo bash install.sh \

  --limit <GB>                     # 例: 170, 200

  --cmd-ydcx <命令名>               # 自定义月度查询命令
  --telegram-bot <token>           # Telegram Bot Token

  --wechat-webhook <url>           # 企业微信 Webhook URL

  --email-to <addr>                # 收件人
```
